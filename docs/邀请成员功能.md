# 邀请成员功能

## 概述

团队管理员通过生成邀请链接（带 token、role、过期时间）邀请成员加入团队。被邀请者完成 OAuth 登录后自动绑定并加入团队。整个过程可撤销、可审计。

**核心特点：**
- ✅ 安全：可过期、可撤销、可追踪
- ✅ 标准 RBAC + 邀请机制
- ✅ 无需邮件服务：支持"复制链接"分享

---

## 邀请链接格式

```
https://yourapp.com/invite?token=xxxx
```

---

## 用户流程

### 管理员视角

1. 在「成员管理」页面点击「邀请成员」
2. 选择被邀请者角色（member / admin）
3. 系统生成邀请链接（包含 token）
4. 复制链接发送给被邀请者（微信/邮件/飞书）

### 被邀请者视角

1. 打开邀请链接 `/invite?token=abc123`
2. 前端校验邀请 token 有效性
3. 点击「使用飞书登录并加入」
4. 完成 OAuth 授权
5. 自动加入团队，跳转到 Dashboard

## 数据模型

### Invite 表

```prisma
model Invite {
  id          String       @id @default(cuid())
  token       String       @unique
  teamId      String
  team        Team         @relation(fields: [teamId], references: [id])
  role        TeamRole     @default(MEMBER)
  status      InviteStatus @default(PENDING)
  invitedBy   String
  inviter     User         @relation("InviterRelation", fields: [invitedBy], references: [id])
  acceptedBy  String?
  accepter    User?        @relation("AccepterRelation", fields: [acceptedBy], references: [id])
  expiresAt   DateTime
  createdAt   DateTime     @default(now())
  updatedAt   DateTime     @updatedAt
}

enum InviteStatus {
  PENDING    // 待使用
  ACCEPTED   // 已接受
  EXPIRED    // 已过期
  REVOKED    // 已撤销
}

enum TeamRole {
  OWNER
  ADMIN
  MEMBER
}
```

## API 设计

### 1. 创建邀请链接

```
POST /api/invites
```

**Request Body:**
```json
{
  "teamId": "team_xxx",
  "role": "MEMBER",
  "expiresInDays": 7
}
```

**Response:**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "invite_xxx",
    "token": "abc123",
    "inviteUrl": "https://yourapp.com/invite?token=abc123",
    "expiresAt": "2026-03-09T00:00:00Z"
  }
}
```

### 2. 校验邀请 Token

```
GET /api/invites/validate?token=abc123
```

**Response (成功):**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "valid": true,
    "team": {
      "id": "team_xxx",
      "name": "My Team"
    },
    "role": "MEMBER",
    "inviter": {
      "name": "张三"
    },
    "expiresAt": "2026-03-09T00:00:00Z"
  }
}
```

**Response (失败):**
```json
{
  "code": 40001,
  "message": "邀请链接已过期"
}
```

### 3. 获取邀请列表（管理员）

```
GET /api/invites?teamId=team_xxx
```

**Response:**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [
      {
        "id": "invite_xxx",
        "token": "abc123",
        "role": "MEMBER",
        "status": "PENDING",
        "inviter": { "name": "张三" },
        "expiresAt": "2026-03-09T00:00:00Z",
        "createdAt": "2026-03-02T00:00:00Z"
      }
    ],
    "total": 1
  }
}
```

### 4. 撤销邀请

```
DELETE /api/invites/:id
```

**Response:**
```json
{
  "code": 0,
  "message": "邀请已撤销"
}
```

## 前端页面

### 邀请页面 `/invite`

| 状态 | 展示内容 |
|------|----------|
| 校验中 | Loading 状态 |
| 有效 | 显示团队名称、邀请人、角色，以及「使用飞书登录并加入」按钮 |
| 无效/过期 | 错误提示 + 返回首页链接 |
| 已登录用户 | 直接加入团队（无需再次 OAuth） |

### 成员管理页面

- 邀请列表：显示所有邀请记录（状态、创建时间、过期时间）
- 生成邀请：选择角色、生成链接、复制
- 撤销邀请：撤销未使用的邀请

## 安全考虑

1. **Token 安全**
   - 使用 cuid/uuid 生成，不可预测
   - 单次使用，接受后立即失效
   - 支持过期时间（默认 7 天）

2. **权限控制**
   - 仅 OWNER / ADMIN 可创建邀请
   - 仅 OWNER 可邀请 ADMIN
   - 不能邀请比自己权限高的角色

3. **审计日志**
   - 记录邀请创建、接受、撤销事件
   - 关联操作人和时间戳

## 实现优先级

| 优先级 | 功能 |
|--------|------|
| P0 | 生成邀请链接、校验 Token、接受邀请 |
| P1 | 邀请列表、撤销邀请 |
| P2 | 邀请过期自动清理、审计日志 |
